# P12 真实使用验证计划

## P12-USAGE-A0：Voice Lab 真实使用流验证方案

**创建时间：** 2026-05-15

---

## 1. 背景与目标

**产品定位：** 本地 Web App / 单用户 AI 音频创作工作台

**验证目的：**
- 确认所有核心功能在真实 MiniMax API 下可正常运行
- 发现 E2E mock 测试无法覆盖的问题（网络延迟、真实 API 响应结构、错误边界）
- 验证产品打磨（B1–B6）后的真实用户体验

**限制：**
- 不改业务代码
- 不引入新的生产问题
- 发现问题后记录，不立即动手修

---

## 2. 前提条件

### 2.1 账号准备

| 项目 | 要求 |
|---|---|
| MiniMax 账号 | 真实账号，有效 API Key |

### 2.2 测试环境

| 项目 | 要求 |
|---|---|
| Browser | Chrome 最新版 / Edge 最新版 |
| Network | 可访问 MiniMax API（国内/海外网络） |
| DevTools | 打开 Network 和 Console 面板 |

### 2.3 记录工具

- DevTools Network 面板截图
- Console 错误记录
- 手动记录问题到 `docs/P12_USAGE_ISSUES.md`（创建该文件）

## 3. 核心验证流程

### 3.1 单条旁白生成（主流程）

**对应 Tab：** workspace

**验证步骤：**

1. **Setup**
   - 配置 Provider 为 minimax（真实 API）
   - 确认 API Key 已配置在环境或 .env

2. **Profile + Voice 绑定**
   - 切换到 voices tab
   - 查询可用音色（选一个系统音色，不需要克隆/设计）
   - 点击"绑定到人设"，绑定到一个新创建的人设
   - 确认绑定成功

3. **Workspace 生成**
   - 回到 workspace tab
   - 选择刚绑定音色的人设
   - 确认 B1 提示显示"当前音色：xxx"
   - 输入短文本（20字以内）
   - 点击"生成"

4. **预期结果**

| 状态 | 预期 |
|---|---|
| 按钮状态 | "生成中…" → "生成"（恢复） |
| 结果区 | 显示音频播放器 |
| 历史记录 | 任务出现在 history tab |
| 最近任务 | B6 卡片显示该任务 |

5. **失败记录**
   - 截图 Network 面板（请求 + 响应）
   - 记录完整错误信息到 P12_USAGE_ISSUES.md

**测试数据：**
```
文本：你好，这是一个测试音频。
人设：任选一个已绑定音色的 profile
```

---

### 3.2 流式生成验证

**对应 Tab：** workspace

**验证步骤：**

1. 重复 3.1 Setup + Profile/Voice
2. 选择"流式"模式（如果 UI 有此选项）
3. 输入短文本
4. 点击"生成"

**预期结果：**
- 音频流式返回，逐步播放
- 不卡顿、无明显延迟

**注意：** 流式为中优先级，可跳过

---

### 3.3 音色试听验证

**对应 Tab：** voices

**验证步骤：**

1. 切换到 voices tab
2. 选择 minimax provider
3. 点击"查询音色"
4. 等待音色列表加载
5. 点击某个音色的"试听"按钮

**预期结果：**
- 音频播放器出现
- 音频可播放（或提示需要更多文本）

**失败记录：**
- 记录音色 ID、provider、错误信息

---

### 3.4 长文本批量验证

**对应 Tab：** longtext

**验证步骤：**

1. 重复 3.1 绑定步骤（如果尚未绑定）
2. 切换到 longtext tab
3. 确认 B3-longtext 提示显示正常
4. 输入长文本（200字以内，避免过高消费）
5. 选择已绑定音色的人设
6. 选择分段策略为"自动"
7. 点击"提交批量任务"

**预期结果：**

| 状态 | 预期 |
|---|---|
| 提交按钮 | "提交中…" → 恢复 |
| 进度面板 | 显示进度条和分段落数 |
| 最终结果 | 合并音频可播放 |
| 历史记录 | 任务出现在 history tab |

**消费估算：**
- 200字长文本，按 auto 分段预估 2-3 段
- 预估成本 < 0.5 元

---

### 3.5 剧本批量验证

**对应 Tab：** script

**验证步骤：**

1. 切换到 script tab
2. 确认 B3-script 每行提示显示正常
3. 添加 2 行台词（每行 10-20 字）
4. 每行选择已绑定音色的人设
5. 点击"提交批量任务"

**预期结果：**
- 多行分别生成，合并为完整音频
- 每行音色由各自人设的绑定音色决定

**注意：** 剧本批量为中优先级，可跳过

---

### 3.6 音色克隆验证

**对应 Tab：** advanced → 声音克隆

**验证步骤：**

1. 切换到 advanced → 声音克隆
2. 上传一个简短的音频文件（5-30秒，越短越好）
3. 选择 provider
4. 点击"克隆"
5. 等待克隆完成（可能需要 1-2 分钟）
6. 在 voices tab 查询并试听克隆音色

**预期结果：**
- 克隆任务提交成功
- 克隆音色出现在音色列表
- 试听可播放

**消费：** 高（克隆通常按秒计费），记录实际消费

**注意：** 克隆为高消费操作，必须有明确预期再执行

---

### 3.7 声音设计验证

**对应 Tab：** advanced → 声音设计

**验证步骤：**

1. 切换到 advanced → 声音设计
2. 输入描述文本
3. 选择 provider
4. 点击"生成"
5. 等待完成，在 voices tab 查询

**注意：** 声音设计为高消费操作，建议跳过或极小量验证

## 5. 验证优先级

| 优先级 | 流程 | 理由 |
|---|---|---|
| P0 | 单条旁白生成 | 核心主流程，80% 用户使用 |
| P0 | 音色绑定 + 提示 | B1/B2 核心打磨点 |
| P0 | 长文本批量 | 高价值功能 |
| P1 | 音色试听 | 验证音色可用性 |
| P2 | 剧本批量 | 中等优先级 |
| P3 | 流式生成 | 可跳过 |
| P3 | 音色克隆 | 高消费，先验证单条再决定 |
| P3 | 声音设计 | 高消费，先验证单条再决定 |

**建议验证顺序：**
1. 单条旁白生成（workspace）
2. 音色绑定 + voices tab 联动
3. 长文本批量
4. （可选）音色克隆（极小量）
