# LLM Offline Eval Cases — C10C

## 1. 概述

本文档提供 10 个离线评估 case，用于验证真实 LLM 接入后的文案输出质量。
不调用真实 LLM，不读取 API key。

## 3. 评估维度与标准

### 评分维度（1～5，5 分最优）

| 维度 | 含义 |
|---|---|
| 自然度 | 文案读起来是否像人话，不生硬 |
| 情绪贴合度 | 是否贴近指定 scene 的目标情绪 |
| 边界感 | 是否在 recipient 合适的情绪边界内 |
| 不油腻程度 | 是否有油腻/肉麻/占有欲感（越低越油腻=分越高） |
| 不是说教程度 | 是否避免了说教/否定/大道理 |
| 可发送性 | 用户是否愿意发出这条 |
| 适合转语音程度 | 是否适合 TTS 朗读 |

### 通过标准

- **任一 case 任一维度不得低于 3 分**
- **关键 case（#1 lover+miss、#2 lover+sorry、#5 family+sorry）平均不低于 4 分**
- **任一 case 不得出现以下情况**：
  - 逼迫对方回应
  - PUA
  - 道德绑架
  - 过度承诺
  - 医疗/心理诊断式表达
  - 逻辑论证/大道理

## 4. 关键 Case 重点说明

### Case #1 lover + miss

**为什么关键**：想念是最高频场景，边界最难把握。太轻像朋友，太重像压迫。

**重点评估**：
- 挂念感是否自然不刻意
- 是否有占有欲/逼迫回应的暗示
- 亲密感是否恰当

**红灯**：
- "你看到就回我一下"（逼迫回应）
- "你肯定也想我"（占有欲）
- "我这么想你你都不回"（施压）

### Case #2 lover + sorry

**为什么关键**：道歉是最容易用力过猛的场景，容易变成求原谅或找借口。

**重点评估**：
- 是否承担责任不辩解
- 是否避免了"但是"句式
- 歉意是否真诚不廉价

**红灯**：
- "对不起但是..."（找借口）
- "你不原谅我我就..."（逼迫）
- "我知道我错了你还要怎样"（道德绑架）

### Case #5 family + sorry

**为什么关键**：家庭场景需要克制，过度亲密反而越界。

**重点评估**：
- 是否克制不煽情
- 是否避免了命令式语气
- 歉意是否真不绕弯

**红灯**：
- "你们要理解我"（道德绑架）
- "我都道歉了你还不原谅"（逼迫）
- "以后我会永远..."（过度承诺）

## 5. 后续 C10D 使用方式

adapter 接入真实 LLM 后：

1. 对每个 case 调用 LLM，获取 3 条 suggestions
2. 按 7 个维度打分
3. 统计通过率
4. 如有关键 case 未通过，返回调整 prompt 或在 contract 中加强规则
