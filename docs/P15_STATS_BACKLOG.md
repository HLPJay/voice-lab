# P15 统计能力待办

## 1. 当前决策

**P15-STATS-B1 暂不实现。统计面板实现延后。本地统计能力保留为后期待办。**

原因：
1. 管理面板（admin.html）已有服务端统计能力，覆盖系统运行、Provider 调用、后端任务统计。
2. P15-B1 规划的是 localStorage 本地最近统计（SampleStore 最多 200 条，ContextStore 最多 50 条），统计范围有限。
3. 当前产品阶段更重要是保持 P14 创作闭环稳定。
4. 统计能力应根据后期产品使用情况再决定是否启动。

## 2. 已完成的统计设计工作

以下设计文档已完成，可作为后续重启本地统计面板的参考：

- `P15_STATS_A0_DESIGN.md`：统计能力设计方案，含 A/B/C 级指标分级、展示结构、数据边界
- `P15_STATS_B1_A0_IMPLEMENTATION_AUDIT.md`：实现前置审查，含 stats_store.js API、stats.js 职责、index.html 修改范围、测试策略

## 3. 管理面板已有统计能力

管理面板位置：`app/static/admin.html`

**后端接口**：
- `/api/admin/stats/summary` — 总览统计
- `/api/admin/stats/daily` — 每日趋势
- `/api/admin/call-logs` — 调用记录与错误

**后端服务**：`app/services/stats_service.py`（VoiceJob / ProviderCallLog / AudioAsset 聚合）

**已有统计指标**：

| 类别 | 指标 |
|---|---|
| 总览 | 总任务数、成功数、失败数、成功率 |
| 字符与音频 | 总字符数、总音频时长 |
| Provider | 各 Provider 调用次数、错误数、错误率 |
| API 分布 | 按 API 类型分布 |
| 每日趋势 | 按天统计任务量/成功率/字符/时长 |
| 调用记录 | 最近调用（Provider / API / 耗时 / 状态） |
| 错误记录 | 最近错误（Provider / 类型 / 消息） |
| Provider 能力 | Provider 能力矩阵 |

**结论**：服务端系统运行统计 / Provider 调用统计 / 后端任务统计已被管理面板覆盖。

## 4. P15 本地统计的限制

P15-B1 原设计数据源：

| 数据源 | 上限 | 代表范围 |
|---|---|---|
| SampleStore（`voice_lab_recent_samples_v1`） | 200 条 | 当前浏览器最近样本 |
| ContextStore（`voice_lab_sample_context_v1`） | 50 条 | 当前浏览器最近上下文 |

**不能代表**：

- 后端全量历史
- 真实音频资产总量
- 真实资产体积
- 真实失败率
- 真实成本
- Provider 全量成功率
- 多设备数据
- 多用户数据
- SaaS 级用量统计

**产品含义**：P15-B1 是"本地最近创作统计"，不是"系统运营统计"。

## 5. 为什么暂不实现 P15-STATS-B1

1. **管理面板已覆盖系统统计**：总任务/成功/失败/Provider/趋势/错误，无需重复实现。
2. **P15-B1 范围有限**：最多 200 条本地样本，不足以反映长期使用模式和全量资产。
3. **当前主线稳定优先**：P14 创作闭环刚完成，保持稳定比加统计更重要。
4. **数据价值待验证**：需要先有高频使用场景，才能判断本地创作统计的实际价值。

## 6. 后期可选方向

### 方向 A：本地创作统计（轻量）

**定位**：本地最近创作行为观察，回答"我最近用哪个模式最多"。

**指标**：
- 最近样本总数 / 今日 / 近 7 天 / 近 30 天
- 单条 / 异步 / 流式 / 多版本 / 试听 / 长文本 / 剧本 分布
- Provider Top / Profile Top
- 有 asset_id / download_url 数及占比
- 总时长 / 平均时长
- context 总数 / longtext 数 / script 数 / 覆盖率

**特点**：前端实现成本低，统计范围限于本地最近样本。

**重启条件**：需要个人最近创作习惯数据时。

### 方向 B：服务端全量统计（重投入）

**定位**：全量系统运营分析。

**指标**：
- 全量资产数量 / 体积 / 时长
- 全量任务成功率 / 失败原因排行榜
- Provider 全量成功率
- 成本估算（需计费规则）
- 字符消耗 / Token 统计
- 多设备同步统计
- 多用户 SaaS 统计

**特点**：需后端数据库、任务记录体系、计费规则支持。

**重启条件**：需要全量历史、成本分析、多设备同步时。

## 7. 重新启动条件

| 场景 | 推荐方向 |
|---|---|
| 需要个人最近创作习惯数据 | 方向 A：本地创作统计 |
| 需要全量资产/成本/失败率/多设备 | 方向 B：服务端全量统计 |

当 Voice Lab 出现持续高频使用，且需要判断使用模式、资产增长、失败原因或成本时，再重新评估启动方向。

## 8. 当前结论

P15-STATS-B1 暂不实现，设计文档保留备查。管理面板已覆盖系统运行统计。后续根据产品发展再决定是否重启本地统计面板或启动服务端全量统计。
