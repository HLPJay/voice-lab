# P14-PRODUCT-B0：SampleSidebar 常驻策略决策

## 1. 背景

P14 完成了"最近样本 → 详情 → 恢复"闭环，SampleSidebar 作为核心入口承担了以下职责：

- 最近样本本地存储（SampleStore + localStorage）
- 侧边栏卡片展示
- 样本播放 / 下载 / 详情 / 复制 / 填入 / 删除
- 长文本一键恢复
- 剧本一键恢复

## 2. 当前 SampleSidebar 作用

SampleSidebar 不是辅助装饰，而是 P14 context restore 闭环的核心入口：

- 用户通过侧边栏查看最近生成的样本
- 通过详情按钮查看长文本 / 剧本完整内容
- 通过恢复按钮将上下文一键填回生产入口
- 所有操作在右侧固定区域完成，无需切换视图

## 3. 不做隐藏 / 折叠 / 过滤的原因

1. **恢复效率优先**：隐藏或过滤会增加到达恢复入口的步骤，与"一键恢复"的产品目标冲突。
2. **状态复杂度**：引入可见性状态会增加 UI 状态管理成本，当前产品阶段收益有限。
3. **本地单用户工作台**：不存在多用户竞争或权限问题，常驻不带来额外风险。
4. **核心入口而非辅助功能**：SampleSidebar 是 P14 闭环的显示层，隐藏它等同于隐藏功能入口。

## 4. 当前固定策略

| 策略 | 当前状态 |
|---|---|
| SampleSidebar 位置 | 右侧常驻 |
| 隐藏 | 不做 |
| 折叠 | 不做 |
| 按 Tab 过滤 | 不做（保持全局最近样本） |
| 跨 Tab storage event 刷新 | 保留 |

## 5. 后续观察项

- 随着样本数量增长，可观察 localStorage 占用趋势
- 如果样本超过 100 条，可考虑分页或懒加载（暂不预设）
- 如果用户反馈侧边栏太宽，可评估宽度可调（暂不预设）
