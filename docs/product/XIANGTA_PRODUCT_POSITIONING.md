# 想Ta了 · 产品定位说明

## 两个层级，两种使命

```
┌─────────────────────────────────────────────────────┐
│            想Ta了 (XiangTa)                          │
│        面向用户的情绪表达产品                          │
│   让说不出口的话，变成有温度的声音                     │
└──────────────────────┬──────────────────────────────┘
                       │ 调用（只通过 gateway）
┌──────────────────────▼──────────────────────────────┐
│            Voice Lab Core                            │
│          音频能力基础设施底座                          │
│  TTS / VoiceProfile / Provider Adapter / History     │
└─────────────────────────────────────────────────────┘
```

**Voice Lab** 是能力层：它关心的是"怎么生成语音"——模型、采样率、声线、Provider、配额管理。

**想Ta了** 是产品层：它关心的是"一个人想对另一个人说什么"——情绪、场景、表达风格、人际关系。

两者代码完全分离。想Ta了不引用 voice_lab 内部实现，只通过 `voice_lab_gateway.py` 调用其公开服务接口。

---

## 想Ta了 是什么

> 一个帮助人把"说不出口的话"整理成有温度的声音短信的工具。

用户路径：

```
选择对象（恋人 / 父母 / 朋友 / 自己）
    → 选择场景（想念 / 道歉 / 感谢 / 安慰 / 晚安）
    → 随手写几句真实的想法（粗糙、零碎都行）
    → AI 整理成 3 个风格版本（克制 / 温柔 / 真诚）
    → 选一条，配上声音
    → 生成一段语音 + 信笺图
    → 保存到本地，自己决定要不要发
```

---

## 想Ta了 不是什么

| 误解 | 真实定位 |
|---|---|
| AI 恋人 / 情感陪伴机器人 | 不是。它不主动说话，不维持对话。|
| 聊天机器人 | 不是。没有多轮对话，没有聊天界面。|
| 自动发送工具 | 不是。生成的内容只保存本地，用户自己决定是否发送。|
| 声音克隆产品 | 不是。使用固定声线预设，不采集用户声音。|
| 情感分析 / 心理咨询 | 不是。只做表达辅助，不做诊断或建议。|
| SaaS / 多用户平台 | MVP 不是。单设备本地体验优先。|

---

## 核心价值主张

1. **说得出来**：把零碎、粗糙的真实想法整理成完整的句子
2. **听起来像自己**：3 种风格，用户选最像自己的那个
3. **有声音的温度**：文字 + 语音双输出，比文字更有温度
4. **不替你做决定**：生成后本地保存，用户自主决定何时、是否发送

---

## 与 Voice Lab 的关系

| 维度 | Voice Lab Core | 想Ta了 |
|---|---|---|
| 用户 | 开发者 / 内部工具 | 普通用户 |
| 概念 | voice_id, model, sample_rate, provider | 对象、场景、风格、声线名字 |
| 接口 | REST + 能力注册表 | 产品语义 API |
| 存储 | PostgreSQL + 文件系统 | 本地 localStorage（MVP） |
| 部署 | 服务端 | H5 / PWA（MVP） |
